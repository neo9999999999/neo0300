## 🤖 텔레그램 알림

### 1️⃣ 봇 만들기 (한 번만)

1. 텔레그램에서 `@BotFather` 검색
2. `/newbot` → 봇 이름 입력 → **TOKEN 받기** (예: `123:abc...`)
3. 본인 텔레그램에서 새 봇 검색 → `/start` 누르기
4. 브라우저에서 https://api.telegram.org/bot{TOKEN}/getUpdates 열기
5. `"chat":{"id":123456789}` 에서 **chat_id 확인**

### 2️⃣ 환경 변수 설정

```bash
# ~/.zshrc 또는 ~/.bashrc 에 추가
export TG_BOT_TOKEN='123:abc...'
export TG_CHAT_ID='123456789'

# 적용
source ~/.zshrc
```

### 3️⃣ 수동 발송 테스트

```bash
cd /Users/neo/Desktop/jongga_picker
source venv/bin/activate

# 메시지만 확인 (발송 X)
python3 telegram_alert.py --dry-run

# 캐시 데이터로 즉시 발송
python3 telegram_alert.py

# 실시간 스캔 후 발송 (5~10분)
python3 telegram_alert.py --live
```

### 4️⃣ 매일 자동 발송 (cron)

```bash
# crontab 편집
crontab -e

# 평일 매일 오후 3:25 KST (장 마감 전)
25 15 * * 1-5 cd /Users/neo/Desktop/jongga_picker && \
    /Users/neo/Desktop/jongga_picker/venv/bin/python3 telegram_alert.py --live \
    >> /Users/neo/Desktop/jongga_picker/logs/tg.log 2>&1
```

또는 macOS launchd / Windows 작업 스케줄러 사용.

## 🔧 트러블슈팅

### Q. "오늘은 없음"만 나옴
A. V/S/A/B 조건이 까다로워서 종종 빈 결과 → 정상. 
   하루 평균: V 0~1 / S 0~1 / A 0~1 / B 0~1 정도.

### Q. 실시간 스캔이 너무 느림
A. 정상 (5~10분). 캐시 데이터로 우선 확인 권장.

### Q. 텔레그램 발송 실패
A. TG_BOT_TOKEN, TG_CHAT_ID 환경변수 다시 확인.
   `echo $TG_BOT_TOKEN` 으로 설정 여부 검증.

### Q. cron이 안 됨
A. `crontab -l` 로 등록 확인. `which python3` 절대경로 사용 필수.
